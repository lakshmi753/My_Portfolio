PROJECT DISCRIPTION

This project is a Todo List app, which is made using Html, Css and Javascript. In which we can schedule ours daily tasks. The app uses "local storage" to keep track of our tasks . The second top heading of the app keeps the count of total number of tasks Todo List is containing at the present moment. After that we have a 'text-area' where we can enter our desired tasks and along with it we have a button 'create task' which when clicked displays our task along with the date and time of the created task. This Todo List app provides the facility of "editing" the task. We can remove the task from the app using 'remove' button but only when the task is marked as "done", this features makes us feel as if someone is reminding us to "complete the task first and then move move on or don't skip it." At the end we have a "clear list" button which gives us the option to clear the whole list at once but first makes sure wheather user really want to clear the list.

PROJECT DISCRIPTION

This project is a popular "Hangman-game", which everyone of us has definitely played once or more. This project is made using Html, Css and Javascript. Here I'm using self-create array of "words and hints" required for the game and randomly getting words along with there hints every time the game starts. Six chances were given for incorrect guess and with each incorrect guess our hangman image also get updated. After losing all the six chances there appears a overlay over which we can see a small pop-up having losing-gif, correct-word and a "play again" button, clicking which we can start the game again. Similarly if we guess a correct word, there appears an overlay over which we can see same pop-up but this time with laughing-gif and play-again button as before. The most facinating thing about this project is that in this project we are creating a keyboard using "String.fromCharCode()" method.
